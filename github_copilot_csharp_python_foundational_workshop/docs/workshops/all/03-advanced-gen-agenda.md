# Copilot Prompt File
## Purpose:
A prompt file is essentially a structured set of instructions that define how Copilot should behave for a specific scenario or task. It’s used to customize the tone, style, and logic of responses without creating a separate agent.

## Key Features:
- Defines system-level instructions for Copilot (e.g., role, personality, constraints).
- Can include examples, context, and rules for handling user queries.
- Applied at runtime to influence Copilot’s behavior for a particular session or domain.


## Use Case:
If you want Copilot to follow a specific workflow (e.g., drafting legal documents, summarizing architecture diagrams) without building a new agent, you use a prompt file.

# Custom Agents


## Purpose:
A custom agent is a fully configured AI assistant tailored for a specific business process or domain. It goes beyond prompt customization by integrating data sources, plugins, and workflows.

## Key Features:

Has its own identity and capabilities (like a specialized bot).
Can connect to enterprise data, APIs, and external systems.
Supports multi-turn conversations with persistent context.
Often includes custom actions (e.g., triggering workflows, fetching data).

## Use Case:
If you need an AI that acts as a dedicated HR assistant, IT helpdesk bot, or project management assistant, you build a custom agent.

# Summary
- Prompt File = Behavior customization for Copilot.
- Custom Agent = A dedicated AI assistant with integrated data and actions.
